Self Hosted Maps
================

Manager command:
  self-hosted-maps-manager

Direct installed path:
  /opt/self-hosted-maps/bin/map-manager.sh

Useful commands:
  self-hosted-maps-manager
  self-hosted-maps-rebuild
  self-hosted-maps-refresh-catalog
  self-hosted-maps-list-installed
  self-hosted-maps-install-imagery
  self-hosted-maps-list-imagery
  self-hosted-maps-remove-imagery
  self-hosted-maps-install-terrain
  self-hosted-maps-update-app

State file:
  /etc/self-hosted-maps/datasets.json

Current served tileset:
  /var/lib/self-hosted-maps/current/openmaptiles.mbtiles

Offline imagery root:
  /var/lib/self-hosted-maps/imagery

Optional terrain artifact:
  /var/lib/self-hosted-maps/current/terrain/terrain-manifest.json

Viewer:
  http://<server-ip>/

Viewer features:
- top-left coordinate or address search
- a Maps modal for browsing installed maps, downloading catalog datasets, and rebuilding the active map selection
- one right-side layer menu for OpenSky, ADS-B Exchange, AIS vessels, TomTom traffic, installed offline raster imagery, local terrain/hillshade, local street imagery, and offline OSM layer toggles
- a disabled-by-default satellite catalog-only status button when cached orbital elements are enabled; v1 does not draw satellite positions

Flight data configuration:
- OpenSky works anonymously by default. For authenticated REST API access, set SHM_OPENSKY_CLIENT_ID and SHM_OPENSKY_CLIENT_SECRET in /etc/self-hosted-maps/self-hosted-maps.runtime.conf.
- SHM_OPENSKY_TOKEN_URL is optional and defaults to OpenSky's OAuth2 token endpoint.
- Restart self-hosted-maps-api after changing runtime flight credentials.

Vessel and traffic data configuration:
- AISStream vessel positions are disabled by default. Set SHM_AISSTREAM_ENABLED=1 and SHM_AISSTREAM_API_KEY in /etc/self-hosted-maps/self-hosted-maps.runtime.conf.
- TomTom traffic flow and incident overlays are disabled by default. Set SHM_TOMTOM_TRAFFIC_ENABLED=1 and SHM_TOMTOM_API_KEY in /etc/self-hosted-maps/self-hosted-maps.runtime.conf.
- Provider keys stay server-side; the browser uses local /api proxy URLs. Restart self-hosted-maps-api after changing runtime live-layer credentials.

Offline imagery configuration:
- Install only raster MBTiles imagery that permits local storage and serving.
- Required install fields are id, name, MBTiles path, attribution, and license name.
- Example: self-hosted-maps-install-imagery naip-local "NAIP local" /path/to/tiles.mbtiles --attribution "Provider attribution" --license-name "License name"
- The API serves /api/imagery, raw /api/imagery/<id>.json TileJSON, and /api/imagery/<id>/<z>/<x>/<y>.<png|jpg|webp> tile bytes.

Terrain configuration:
- Terrain is optional and served from /var/lib/self-hosted-maps/current/terrain through nginx /terrain/.
- Build small local raster-dem tile sets with python3 /opt/self-hosted-maps/bin/build-terrain-tiles.py and apt-provided GDAL bindings.
- Install prebuilt Terrarium or Mapbox Terrain-RGB-compatible raster-dem PNG tiles with self-hosted-maps-install-terrain.
- Terrain is shown only when terrain-manifest.json matches the current selected_hash and dataset_ids.
- Contours are deferred in this version and reported as unavailable.

Local street imagery configuration:
- Local street imagery is disabled by default. Set SHM_STREET_IMAGERY_ENABLED=1 and optionally SHM_STREET_IMAGERY_ROOT in /etc/self-hosted-maps/self-hosted-maps.runtime.conf.
- The default root is ${SHM_DATA_ROOT}/street-imagery and expects index.json plus relative media paths under that root. Custom SHM_STREET_IMAGERY_ROOT values must still resolve under SHM_DATA_ROOT.
- Public viewer APIs expose only publishable/redacted records and API media URLs; they do not expose local paths.
- Street imagery admin operations require SHM_ADMIN_TOKEN to be set and supplied with the request.
- Panoramax is deferred/third-party-only. Google and Mapillary offline imagery are not implemented.

Satellite element cache configuration:
- Satellite orbital element catalog support is disabled by default and cache/catalog only. Set SHM_SATELLITES_ENABLED=1 to expose cached OMM metadata in the local API and viewer status button.
- CelesTrak refresh and local OMM import write under /var/lib/self-hosted-maps/cache/satellites by default and require SHM_ADMIN_TOKEN when triggered through /api/admin/satellites/refresh or /api/admin/satellites/import.
- Space-Track and orbital propagation are not included in v1.

Manager capabilities:
- browse the live catalog
- install additional datasets over time
- filter installed datasets by id, name, provider, or status
- inspect installed dataset details
- check for dataset updates
- redownload a dataset via the update action
- install, list, and remove licensed offline raster imagery overlays with direct commands
- choose selected datasets for the next rebuild
- rebuild the served map explicitly
- update the installed application from a local source checkout

Application updates:
- Run self-hosted-maps-manager and choose "Update application", or run self-hosted-maps-update-app directly.
- The updater applies an already-updated local checkout into /opt/self-hosted-maps. It does not run git pull, update datasets, rebuild maps, install packages, or change runtime secrets.
- By default, app updates refresh installed scripts, viewer assets, the repo-owned Tilemaker profile, manager docs, and command shortcuts while preserving downloaded datasets, current map tiles, runtime credentials, dataset state, and /opt/self-hosted-maps/www/vendor.
- Rebuild the served map after Tilemaker profile changes so current MBTiles include the new schema.
- Advanced system config refresh is optional and separately confirmed because it can update systemd, nginx, and TileServer config.
- App update metadata is written to /etc/self-hosted-maps/app-manifest.json. Backups are stored under /var/lib/self-hosted-maps/backups/app-update.

Notes:
- The installer bootstraps the first dataset into the dataset state automatically.
- The manager shows a one-time first-run hint when the system has only the bootstrap dataset installed.
- Rebuilding the served map is explicit; installing or selecting datasets does not automatically rebuild.
- `current.dataset_ids` in the state file tracks the dataset ids that were actually part of the last successful rebuild.
- The scheduled maintenance job refreshes the datasets that back the current served map instead of rebuilding from the legacy single bootstrap URL.
